# CUDA Launch Setup APIs

Kernel launch is a system-level mechanism exposed through the device runtime library, allowing kernels to be launched directly from device code [CUDA_C_Programming_Guide:L14022-L14043]. This functionality is available directly from PTX via the underlying `cudaGetParameterBuffer()` and `cudaLaunchDevice()` APIs [CUDA_C_Programming_Guide:L14022-L14043].

While the host-side `<<<>>>` syntax maps to host runtime launch APIs, device-side code uses these specific launch setup APIs [CUDA_C_Programming_Guide:L14022-L14043]. The `<<<>>>` operator is translated by the compiler front-end into calls to these functions, enabling users targeting PTX to enact launches [CUDA_C_Programming_Guide:L14022-L14043].

## API Declarations

The APIs for these launch functions differ from the standard CUDA Runtime API [CUDA_C_Programming_Guide:L14022-L14043]. They are defined as follows:

### cudaGetParameterBuffer

`cudaGetParameterBuffer()` allocates and returns a pointer to a parameter buffer [CUDA_C_Programming_Guide:L14090-L14113].

**CUDA-level Declaration:**
```c
extern "C" __device__
void *cudaGetParameterBuffer(size_t alignment, size_t size);
```

**PTX-level Declaration (64-bit address space):**
```txt
extern .func(.param .b64 func_retval0) cudaGetParameterBuffer
(
  .param .b64 alignment,
  .param .b64 size
)
;
```

The first parameter specifies the alignment requirement of the parameter buffer, and the second parameter specifies the size requirement in bytes [CUDA_C_Programming_Guide:L14090-L14113]. In the current implementation, the returned buffer is always guaranteed to be 64-byte aligned, and the alignment parameter is ignored [CUDA_C_Programming_Guide:L14090-L14113]. However, it is recommended to pass the correct alignment requirement (the largest alignment of any parameter to be placed in the buffer) to ensure future portability [CUDA_C_Programming_Guide:L14090-L14113].

### cudaLaunchDevice

`cudaLaunchDevice()` executes the kernel using the provided configuration and parameter buffer [CUDA_C_Programming_Guide:L14058-L14089].

**CUDA-level Declaration:**
```c
extern "C" __device__
cudaError_t cudaLaunchDevice(void *func, void *parameterBuffer,
                       dim3 gridDimension, dim3 blockDimension,
                       unsigned int sharedMemSize,
                       cudaStream_t stream);
```

**PTX-level Declaration (64-bit address space):**
```txt
.extern .func(.param .b32 func_retval0) cudaLaunchDevice
(
    .param .b64 func,
    .param .b64 parameterBuffer,
    .param .align 4 .b8 gridDimension[12],
    .param .align 4 .b8 blockDimension[12],
    .param .b32 sharedMemSize,
    .param .b64 stream
)
;
```

The function is defined in the `cudadevrt` system library, which must be linked with a program to use device-side kernel launch functionality [CUDA_C_Programming_Guide:L14058-L14089].

*   `func`: A pointer to the kernel to be launched [CUDA_C_Programming_Guide:L14058-L14089].
*   `parameterBuffer`: The buffer holding the actual parameters to the launched kernel [CUDA_C_Programming_Guide:L14058-L14089].
*   `gridDimension`, `blockDimension`, `sharedMemSize`, `stream`: Specify the launch configuration [CUDA_C_Programming_Guide:L14058-L14089].

## Parameter Buffer Layout

When populating the parameter buffer, users are responsible for correctly formatting all necessary data structures according to specification [CUDA_C_Programming_Guide:L14022-L14043]. Backwards compatibility is guaranteed for these data structures [CUDA_C_Programming_Guide:L14022-L14043].

Key constraints for the parameter buffer include:

1.  **No Reordering:** Parameter reordering in the parameter buffer is prohibited [CUDA_C_Programming_Guide:L14114-L14119].
2.  **Alignment:** Each individual parameter must be aligned. Specifically, each parameter must be placed at the $n^{th}$ byte in the buffer, where $n$ is the smallest multiple of the parameter size that is greater than the offset of the last byte taken by the preceding parameter [CUDA_C_Programming_Guide:L14114-L14119].
3.  **Size Limit:** The maximum size of the parameter buffer is 4KB [CUDA_C_Programming_Guide:L14114-L14119].

For detailed descriptions of the PTX code generated by the CUDA compiler, refer to the PTX-3.5 specification [CUDA_C_Programming_Guide:L14114-L14119].

## Usage Notes

It is permitted for a CUDA application to call these APIs itself, with the same requirements as for PTX [CUDA_C_Programming_Guide:L14022-L14043]. In both cases, the user is responsible for correctly populating all necessary data structures in the correct format according to specification [CUDA_C_Programming_Guide:L14022-L14043].
