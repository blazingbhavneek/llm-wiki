# CUDA Dynamic Parallelism (CDP1) Parameter Buffer Layout

This page details the layout rules for parameters passed via the parameter buffer in CUDA Dynamic Parallelism version 1 (CDP1). For the CDP2 version of these rules, refer to the CDP2 Parameter Buffer Layout documentation.

## Layout Rules

When constructing the parameter buffer for a CDP1 kernel launch, the following constraints apply:

1. **No Reordering**: Parameter reordering within the parameter buffer is prohibited. Parameters must be placed in the order they are defined or provided.
2. **Alignment**: Each individual parameter must be aligned. Specifically, each parameter must be placed at the $n^{\text{th}}$ byte in the parameter buffer, where $n$ is the smallest multiple of the parameter's size that is greater than the offset of the last byte taken by the preceding parameter.
3. **Size Limit**: The maximum size of the parameter buffer is 4KB.

## Additional Information

For a more detailed description of the PTX code generated by the CUDA compiler for these operations, please refer to the PTX-3.5 specification.
