# CUDA Runtime

The CUDA Runtime is implemented in the `cudart` library, which is linked to the application either statically (via `cudart.lib` or `libcudart.a`) or dynamically (via `cudart.dll` or `libcudart.so`) [[CUDA_C_Programming_Guide:L1211-L1235]]. Applications requiring dynamic linking typically include the corresponding `.dll` or `.so` files as part of their installation package [[CUDA_C_Programming_Guide:L1211-L1235]]. It is only safe to pass the address of CUDA runtime symbols between components that link to the same instance of the CUDA runtime [[CUDA_C_Programming_Guide:L1211-L1235]]. All entry points in the CUDA Runtime API are prefixed with `cuda` [[CUDA_C_Programming_Guide:L1211-L1235]].

The CUDA programming model assumes a system composed of a host and a device, each with their own separate memory [[CUDA_C_Programming_Guide:L1211-L1235]]. The runtime provides functions and concepts to manage this heterogeneous environment, including:

*   **Device Memory**: Functions used to manage device memory [[CUDA_C_Programming_Guide:L1211-L1235]].
*   **Shared Memory**: Mechanisms to maximize performance using shared memory introduced in the thread hierarchy [[CUDA_C_Programming_Guide:L1211-L1235]].
*   **Page-Locked Host Memory**: Host memory that is required to overlap kernel execution with data transfers between host and device memory [[CUDA_C_Programming_Guide:L1211-L1235]].
*   **Asynchronous Concurrent Execution**: Concepts and APIs to enable asynchronous concurrent execution at various levels in the system [[CUDA_C_Programming_Guide:L1211-L1235]].
*   **Multi-Device Systems**: Extensions to the programming model for systems with multiple devices attached to the same host [[CUDA_C_Programming_Guide:L1211-L1235]].
*   **Error Checking**: Methods to properly check errors generated by the runtime [[CUDA_C_Programming_Guide:L1211-L1235]].
*   **Call Stack**: Functions used to manage the CUDA C++ call stack [[CUDA_C_Programming_Guide:L1211-L1235]].
*   **Texture and Surface Memory**: Memory spaces that provide an alternative way to access device memory and expose a subset of the GPU texturing hardware [[CUDA_C_Programming_Guide:L1211-L1235]].
*   **Graphics Interoperability**: Functions to interoperate with the two main graphics APIs, OpenGL and Direct3D [[CUDA_C_Programming_Guide:L1211-L1235]].
