# Memory Reuse Policies

In order to service an allocation request, the driver attempts to reuse memory that was previously freed via `cudaFreeAsync()` before attempting to allocate more memory from the OS [CUDA_C_Programming_Guide:L15607-L15612]. For example, memory freed in a stream can immediately be reused for a subsequent allocation request in the same stream [CUDA_C_Programming_Guide:L15607-L15612]. Similarly, when a stream is synchronized with the CPU, the memory that was previously freed in that stream becomes available for reuse for an allocation in any stream [CUDA_C_Programming_Guide:L15607-L15612].

The stream ordered allocator has a few controllable allocation policies. The pool attributes `cudaMemPoolReuseFollowEventDependencies`, `cudaMemPoolReuseAllowOpportunistic`, and `cudaMemPoolReuseAllowInternalDependencies` control these policies [CUDA_C_Programming_Guide:L15607-L15612]. Upgrading to a newer CUDA driver may change, enhance, augment and/or reorder the reuse policies [CUDA_C_Programming_Guide:L15607-L15612].
