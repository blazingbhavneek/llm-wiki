# Execution Environments

Execution environments are a fundamental concept in the device-side synchronization model, used to encapsulate work and manage completion states for graphs and streams.

## Graph Execution Environments

When a graph is launched from the device, it is launched into its own execution environment. This environment encapsulates all work contained within the graph as well as any "fire and forget" work generated by that graph [CUDA_C_Programming_Guide:L2977-L2997].

A graph is considered complete only when two conditions are met:
1. The graph has completed its execution.
2. All generated child work (fire-and-forget work) is complete [CUDA_C_Programming_Guide:L2977-L2997].

These environments are hierarchical, meaning a graph environment can include multiple levels of child environments resulting from nested fire-and-forget launches [CUDA_C_Programming_Guide:L2977-L2997].

## Stream Environments

When a graph is launched from the host, a stream environment is created that parents the execution environment of the launched graph [CUDA_C_Programming_Guide:L2977-L2997].

The stream environment encapsulates all work generated as part of the overall launch. The stream launch is considered complete—meaning downstream dependent work may now run—when the overall stream environment is marked as complete [CUDA_C_Programming_Guide:L2977-L2997].

## Visualizing the Hierarchy

The relationship between these environments can be visualized as follows:
*   **Fire-and-Forget Launch:** Shows how a graph environment encapsulates child environments from fire-and-forget operations [CUDA_C_Programming_Guide:L2977-L2997].
*   **Nested Environments:** Illustrates multiple levels of child environments within a graph [CUDA_C_Programming_Guide:L2977-L2997].
*   **Stream Environment:** Visualizes the stream environment as the parent to the graph's execution environment [CUDA_C_Programming_Guide:L2977-L2997].
