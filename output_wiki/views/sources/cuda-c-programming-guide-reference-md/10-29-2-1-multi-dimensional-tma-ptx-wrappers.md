# 10.29.2.1 Multi-dimensional TMA PTX wrappers

Part of [Cuda C Programming Guide Reference](README.md). Source lines L10564-L11088.

- [TMA Alignment and Bounds Handling](../../../concepts/tma-alignment-and-bounds.md) — Rules for out-of-bounds handling (zero-filling shared memory for reads, no negative indices for writes) and alignment/stride requirements for global and shared memory in Compute Capability 9.0 TMA operations.
- [CUDA TMA PTX Wrappers (cp.async.bulk.tensor)](../../../entities/cp-async-bulk-tensor-wrappers.md) — C++ wrapper functions for the PTX `cp.async.bulk.tensor` instructions, providing interfaces for 1D through 5D bulk tensor asynchronous copies between global and shared memory.
- [TMA Swizzle for Shared Memory Bank Conflict Avoidance](../../../concepts/tma-swizzle.md) — TMA swizzle patterns (None, 32B, 64B, 128B) rearrange data in shared memory during Tensor Memory Accelerator (TMA) transfers to eliminate bank conflicts, requiring specific alignment and access patterns.
- [TMA Swizzle Modes and Requirements](../../../concepts/tma-swizzle-modes-and-requirements.md) — TMA swizzle patterns map 16-byte chunks of global memory to shared memory banks to avoid bank conflicts, requiring specific alignment and inner dimension constraints based on the swizzle width.
- [Device-side Tensor Map Encoding](../../../concepts/tma-device-encoding.md) — Process for encoding a tiled-type tensor map on the device using a host-created template, modifying it in shared memory, and copying it back to global memory.
- [cuda::ptx::tensormap_replace Functions](../../../entities/cuda-ptx-tensormap-replace.md) — Functions wrapping the tensormap.replace PTX instruction to modify fields (address, rank, box dim, global dim, stride, element size, type, interleave, swizzle, fill mode) of a tiled-type tensor map on device.
- [TMA Tensor Map Release-Acquire Pattern](../../../concepts/tma-release-acquire-pattern.md) — Synchronization requirements for using a modified tensor map in global memory, involving `tensormap_cp_fenceproxy` (release) and `fence_proxy_tensormap_generic` (acquire) to ensure visibility and ordering between threads.
- [make_tensorsmap_template](../../../entities/make-tensorsmap-template.md) — An example function demonstrating how to create a minimal tiled-type tensor map template using the cuTensorMapEncodeTiled Driver API function, which can then be modified on device.
