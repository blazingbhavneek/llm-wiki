# Chapter 1. Overview

Part of [Cuda C Programming Guide Reference](README.md). Source lines L530-L1140.

- [CUDA C++ Language Support](../../../concepts/cuda-c-language-support.md) — CUDA C++ language support provides a mix of host and device code compilation via nvcc, supporting C++11, C++14, C++17, and C++20 features with specific restrictions on standard libraries, exceptions, and device-specific attributes.
- [CUDA Compute Capabilities](../../../concepts/cuda-compute-capabilities.md) — CUDA Compute Capability is a versioning scheme (X.Y) that identifies GPU hardware features, architecture generations, and instruction sets available to applications at runtime.
- [CUDA Driver API](../../../concepts/cuda-driver-api.md) — The CUDA Driver API is a low-level C API that provides direct control over CUDA contexts and modules, serving as the foundation for the higher-level CUDA Runtime API.
- [CUDA Error Log Management](../../../concepts/cuda-error-log-management.md) — CUDA Error Log Management provides mechanisms for activating, configuring, and retrieving error logs through specific APIs, while operating within defined limitations.
- [Unified Memory Programming](../../../concepts/unified-memory.md) — Unified Memory Programming provides a single, system-wide address space for CPU and GPU memory, managed via APIs like cudaMallocManaged and __managed__ variables, with support levels varying by device architecture.
- [CUDA Lazy Loading](../../../concepts/cuda-lazy-loading.md) — CUDA Lazy Loading is a feature that defers the loading of device code until it is actually needed, improving startup time and reducing memory usage, with specific version requirements for drivers, toolkits, and compilers.
- [Extended GPU Memory (EGM)](../../../concepts/extended-gpu-memory.md) — Extended GPU Memory (EGM) is a feature for integrated CPU-GPU NVIDIA systems that allows GPU threads to efficiently access all system memory, including CPU-attached memory and HBM3, via high-bandwidth interconnects like NVLink-C2C and NVSwitch.
- [CUDA Overview](../../../concepts/cuda-overview.md) — CUDA is a parallel computing platform and programming model developed by NVIDIA that enables GPU acceleration for compute-intensive applications using C, C++, and Fortran.
- [CUDA Programming Model](../../../concepts/cuda-programming-model.md) — The CUDA parallel programming model is a scalable architecture that leverages multicore CPUs and manycore GPUs through thread group hierarchies, shared memory, and barrier synchronization.
- [CUDA Kernels](../../../concepts/cuda-kernels.md) — CUDA kernels are C++ functions defined with the __global__ specifier that execute in parallel across multiple CUDA threads, launched using the <<<...>>> execution configuration syntax.
- [CUDA Thread Hierarchy](../../../concepts/cuda-thread-hierarchy.md) — The CUDA thread hierarchy organizes threads into blocks and grids using 1D, 2D, or 3D indices to map parallel computations to data domains.
- [CUDA Shared Memory](../../../concepts/cuda-shared-memory.md) — CUDA shared memory is a low-latency memory space enabling data sharing and cooperation among threads within a block, typically synchronized using the __syncthreads() barrier.
- [CUDA Thread Block Clusters](../../../concepts/cuda-thread-block-clusters.md) — Thread Block Clusters, introduced in Compute Capability 9.0, group up to eight thread blocks to guarantee co-scheduling on a single GPU Processing Cluster (GPC) and enable hardware-supported synchronization via Distributed Shared Memory.
- [CUDA Memory Hierarchy](../../../concepts/cuda-memory-hierarchy.md) — The CUDA memory hierarchy consists of private local memory, block-scoped shared memory, and globally accessible global, constant, and texture memory spaces, each optimized for different access patterns.
- [CUDA Heterogeneous Programming](../../../concepts/cuda-heterogeneous-programming.md) — CUDA heterogeneous programming models the GPU as a coprocessor with separate memory spaces from the CPU host, unified by Managed Memory.
- [CUDA Asynchronous Programming](../../../concepts/cuda-asynchronous-programming.md) — An introduction to the asynchronous programming model for memory operations, starting with Ampere architecture.
- [CUDA Programming Interface](../../../concepts/cuda-programming-interface.md) — The CUDA Programming Interface enables C++ developers to write device-executed programs through a minimal set of language extensions and a runtime library.
- [CUDA NVCC Compiler](../../../concepts/cuda-nvcc.md) — nvcc is the compiler driver for CUDA that simplifies the compilation of C++ and PTX code into binary form for device execution.
