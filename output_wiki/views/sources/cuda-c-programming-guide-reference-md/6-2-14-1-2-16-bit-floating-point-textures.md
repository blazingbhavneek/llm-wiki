# 6.2.14.1.2 16-Bit Floating-Point Textures

Part of [Cuda C Programming Guide Reference](README.md). Source lines L3811-L4249.

- [16-Bit Floating-Point Textures](../../../concepts/half-precision-textures.md) — Details on the 16-bit floating-point (half) format in CUDA, its IEEE 754-2008 binary2 compatibility, conversion intrinsics, promotion to 32-bit during fetch, and channel description creation.
- [Layered Textures](../../../concepts/layered-textures.md) — Layered textures are one- or two-dimensional texture arrays supported on compute capability 2.0+ devices, created via cudaMalloc3DArray with the cudaArrayLayered flag and accessed using tex1DLayered or tex2DLayered functions.
- [Cubemap Textures](../../../concepts/cubemap-textures.md) — Cubemap textures are six-layered 2D textures addressed by direction vectors, supported on compute capability 2.0+ devices via specific CUDA array and texture fetch APIs.
- [Cubemap Layered Textures](../../../concepts/cubemap-layered-textures.md) — Cubemap layered textures are layered CUDA arrays where each layer is a cubemap of the same dimension, addressed via an integer layer index and three floating-point coordinates, supported on compute capability 2.0+ devices.
- [Texture Gather](../../../concepts/texture-gather.md) — Texture gather is a special 2D texture fetch operation performed by tex2Dgather() that returns four 32-bit values corresponding to a specific component of the four texels used for bilinear filtering, subject to 8-bit fractional precision limits and specific hardware and memory requirements.
- [Surface Memory](../../../concepts/surface-memory.md) — Surface memory allows read and write access to CUDA arrays via surface objects on devices with compute capability 2.0 and higher.
- [Surface Object API](../../../concepts/surface-object-api.md) — The Surface Object API allows creating surface objects from CUDA arrays using cudaCreateSurfaceObject(), utilizing byte addressing for read/write operations distinct from texture memory.
- [Cubemap Surfaces](../../../concepts/cubemap-surfaces.md) — Cubemap surfaces are accessed as two-dimensional layered surfaces using surfCubemapread and surfCubemapwrite, where an integer index selects the face and floating-point coordinates address the texel.
- [Cubemap Layered Surfaces](../../../concepts/cubemap-layered-surfaces.md) — Cubemap layered surfaces are accessed as two-dimensional layered surfaces using specific read and write functions with face indices and texture coordinates.
- [CUDA Arrays](../../../concepts/cuda-arrays.md) — CUDA arrays are opaque, dimensionally flexible memory layouts optimized for texture fetching, accessible only via texture or surface references.
- [Read/Write Coherency](../../../concepts/read-write-coherency.md) — Texture and surface memory are cached and not kept coherent with global memory writes within the same kernel call, meaning reads to addresses written to in the same kernel return undefined data.
- [Graphics Interoperability](../../../concepts/graphics-interoperability.md) — Graphics interoperability allows mapping OpenGL and Direct3D resources into the CUDA address space for data exchange, requiring explicit registration, mapping, and pointer access while preventing concurrent access.
- [OpenGL Interoperability](../../../concepts/opengl-interoperability.md) — Details on interoperating with OpenGL buffers, textures, and renderbuffers, including registration functions, usage hints, bindless texture limitations, and a code sample for modifying vertex buffers.
- [Direct3D Interoperability](../../../concepts/direct3d-interoperability.md) — Overview of Direct3D 9Ex, 10, and 11 interoperability, device criteria, resource registration, and reference to specific version sections.
- [Direct3D 9 Interoperability](../../../concepts/direct3d-9-interoperability.md) — Direct3D 9 interoperability allows CUDA kernels to map and modify Direct3D 9 vertex buffers, enabling dynamic vertex manipulation by integrating Direct3D device initialization with CUDA resource registration and mapping.
