# 6.2.15.2.2 Direct3D 10 Version

Part of [Cuda C Programming Guide Reference](README.md). Source lines L4250-L4737.

- [CUDA Direct3D 10 Interoperability](../../../concepts/cuda-direct3d-10-interoperability.md) — CUDA Direct3D 10 interoperability allows sharing graphics resources between Direct3D 10 and CUDA by registering, mapping, and unregistering resources to enable GPU-accelerated data processing.
- [CUDA Direct3D 11 Interoperability](../../../concepts/cuda-direct3d-11-interoperability.md) — CUDA Direct3D 11 interoperability allows sharing resources between CUDA and Direct3D 11 by registering D3D11 resources with the CUDA graphics API, mapping them for kernel access, and unregistering them upon completion.
- [CUDA SLI Interoperability](../../../concepts/cuda-sli-interoperability.md) — CUDA SLI interoperability involves specific memory and context management considerations when integrating CUDA with Direct3D or OpenGL in multi-GPU SLI configurations.
- [CUDA External Resource Interoperability](../../../concepts/cuda-external-resource-interoperability.md) — CUDA External Resource Interoperability enables the import of external memory objects and synchronization semaphores from other APIs into the CUDA environment for device access and execution control.
- [CUDA Vulkan Interoperability](../../../concepts/cuda-vulkan-interoperability.md) — CUDA Vulkan Interoperability allows applications to share memory and synchronization objects between CUDA and Vulkan by matching device UUIDs, importing memory via various handle types, and mapping buffers or mipmapped arrays onto the imported memory.
