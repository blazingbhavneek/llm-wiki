# Chapter 26. Extended GPU Memory

Part of [Cuda C Programming Guide Reference](README.md). Source lines L22200-L22427.

- [Extended GPU Memory (EGM)](../../../concepts/extended-gpu-memory.md) — Extended GPU Memory (EGM) is a feature for integrated CPU-GPU NVIDIA systems that allows GPU threads to efficiently access all system memory, including CPU-attached memory and HBM3, via high-bandwidth interconnects like NVLink-C2C and NVSwitch.
- [EGM Preliminaries](../../../concepts/egm-preliminaries.md) — An overview of the prerequisites for Extended GPU Memory (EGM), including supported topologies, identifier assignment, virtual memory management requirements, and relevant CUDA types.
- [EGM Platforms and System Topology](../../../concepts/egm-platforms-topology.md) — Extended GPU Memory (EGM) supports three system topologies: Single-Node Single-GPU, Single-Node Multi-GPU, and Multi-Node Single-GPU, with specific device management requirements.
- [EGM Socket Identifiers (NUMA)](../../../concepts/egm-socket-identifiers.md) — EGM utilizes operating system-assigned NUMA node identifiers to manage memory placement, accessible via the CU_DEVICE_ATTRIBUTE_HOST_NUMA_ID device attribute.
- [EGM Allocators](../../../concepts/egm-allocators.md) — Extended GPU Memory (EGM) supports specific memory allocators that leverage NVLink routing for optimized access to system memory across NUMA nodes.
- [EGM Memory Management APIs](../../../concepts/egm-memory-management-apis.md) — Extensions to current CUDA APIs for memory management in EGM, including support for Virtual Memory and Stream Ordered Memory allocators with NUMA-aware location properties.
- [Using the EGM Interface](../../../concepts/egm-interface-usage.md) — This section covers the practical usage of the Extended GPU Memory (EGM) interface within the CUDA C programming model.
- [EGM: Single-Node, Single-GPU](../../../concepts/egm-single-node-single-gpu.md) — In a single-node, single-GPU configuration, existing CUDA host allocators and system-allocated memory can be utilized to leverage high-bandwidth CPU-to-CPU (C2C) communication, where local access behaves as it does for standard host allocations.
- [EGM: Single-Node, Multi-GPU](../../../concepts/egm-single-node-multi-gpu.md) — EGM in a single-node, multi-GPU configuration requires users to provide host placement information via NUMA node IDs, which can be retrieved using CUDA device attributes, before allocating memory through VMM or CUDA Memory Pool.
- [EGM: Using VMM APIs](../../../concepts/egm-vmm-usage.md) — This page details the usage of Virtual Memory Management (VMM) APIs for Explicit Global Memory (EGM) allocations, covering physical memory creation, address space reservation, mapping, and access protection.
- [EGM: Using CUDA Memory Pool](../../../concepts/egm-cuda-memory-pool-usage.md) — This page details the configuration and usage of CUDA Memory Pool APIs to create and manage Explicit Graph Memory (EGM) with peer access capabilities.
- [EGM: Multi-Node, Single-GPU](../../../concepts/egm-multi-node-single-gpu.md) — This page details the configuration and memory allocation procedures for using the CUDA Extensible Graphics Memory (EGM) API in a multi-node, single-GPU environment, where memory is allocated on one node's NUMA domain and shared with another node via fabric handles.
