# 10.25. DPX

Part of [Cuda C Programming Guide Reference](README.md). Source lines L9083-L9777.

- [DPX (Data Parallel eXtensions)](../../../concepts/dpx.md) — DPX is a set of hardware-accelerated functions available on compute capability 9+ devices for performing min/max operations, fused addition/min/max, and optional ReLU clamping on integer parameters, primarily used in dynamic programming algorithms.
- [Asynchronous Barrier (cuda::barrier)](../../../concepts/asynchronous-barrier.md) — The NVIDIA C++ standard library provides a GPU implementation of std::barrier via cuda::barrier, allowing for asynchronous arrive/wait synchronization with hardware acceleration on compute capability 8.0+.
- [Memory Barrier Primitives (__mbarrier)](../../../concepts/memory-barrier-primitives.md) — C-like interfaces to cuda::barrier functionality available via <cuda_awbarrier_primitives.h>, providing data types and API functions for managing shared memory synchronization phases.
- [Asynchronous Data Copies (memcpy_async)](../../../concepts/async-data-copies.md) — Asynchronous Data Copies (memcpy_async) is a CUDA 11+ feature that allows device code to explicitly manage asynchronous copying of data, enabling the overlap of computation with data movement through APIs in cuda/barrier, cuda/pipeline, and cooperative_groups/memcpy_async.h.
