# 6.1.1.2 Just-in-Time Compilation

Part of [Cuda C Programming Guide Reference](README.md). Source lines L1141-L1850.

- [Just-in-Time Compilation](../../../concepts/just-in-time-compilation.md) — Just-in-Time (JIT) compilation is the process where the CUDA device driver compiles PTX code to binary code at runtime, allowing applications to benefit from driver improvements and run on newer hardware, with the resulting binaries cached in the compute cache.
- [Binary Compatibility](../../../concepts/binary-compatibility.md) — Binary code (cubin) is architecture-specific and guaranteed to be compatible from one minor revision to the next (e.g., X.y to X.z where z >= y), but not across major revisions or to previous minor revisions, with support limited to desktop platforms.
- [PTX Compatibility](../../../concepts/ptx-compatibility.md) — PTX compatibility is determined by the -arch compiler option, with architecture-specific targets being incompatible across generations and family-specific targets offering forward compatibility within a family.
- [Application Compatibility](../../../concepts/application-compatibility.md) — Application compatibility in CUDA is managed by embedding binary and PTX code via compiler options, allowing runtime selection or JIT compilation based on device compute capability.
- [C++ Compatibility](../../../concepts/cpp-compatibility.md) — The CUDA compiler front end processes source files according to C++ syntax rules, supporting full C++ for host code but only a subset for device code.
- [64-Bit Compatibility](../../../concepts/64-bit-compatibility.md) — The 64-bit version of nvcc compiles device code in 64-bit mode, which requires that host code also be compiled in 64-bit mode.
- [CUDA Runtime](../../../concepts/cuda-runtime.md) — The CUDA Runtime is implemented in the cudart library and provides the host-device memory model, asynchronous execution, error checking, and graphics interoperability APIs.
- [Runtime Initialization](../../../concepts/runtime-initialization.md) — As of CUDA 12.0, cudaInitDevice() and cudaSetDevice() explicitly initialize the CUDA runtime and primary context, triggering JIT compilation and requiring error checking, while implicit initialization occurs on device 0 if no explicit call is made.
- [Device Memory](../../../concepts/device-memory.md) — Device memory is the separate memory space on a GPU where kernels operate, allocated as linear memory (unified address space) or CUDA arrays (optimized for textures), with data transfer managed via specific runtime API functions.
- [L2 Cache Management](../../../concepts/l2-cache-management.md) — L2 Cache Management allows applications on CUDA compute capability 8.0+ devices to designate portions of the L2 cache for persisting data accesses, improving bandwidth and latency for repeated global memory accesses through set-aside sizing, access policy windows, and explicit reset mechanisms.
- [Shared Memory](../../../concepts/shared-memory.md) — Shared memory is a fast, software-managed scratchpad memory allocated with the __shared__ specifier that minimizes global memory accesses by allowing threads within a block to cooperate and reuse data.
