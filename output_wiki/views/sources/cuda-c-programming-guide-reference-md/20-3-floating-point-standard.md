# 20.3. Floating-Point Standard

Part of [Cuda C Programming Guide Reference](README.md). Source lines L19524-L20154.

- [CUDA Floating-Point Standard](../../../concepts/cuda-floating-point-standard.md) — CUDA compute devices adhere to the IEEE 754-2008 standard for binary floating-point arithmetic with specific deviations in rounding, exception handling, NaN behavior, and atomic operations.
- [Compute Capability 5.x](../../../entities/cuda-compute-capability-5.x.md) — Compute Capability 5.x devices, based on the Maxwell architecture, feature SMs with 128 CUDA cores, 32 SFUs, and 4 warp schedulers, along with a flexible memory hierarchy including configurable L1/texture cache and shared memory sizes.
- [Compute Capability 6.x](../../../entities/cuda-compute-capability-6.x.md) — Compute Capability 6.x devices, part of the Pascal architecture, feature SMs with varying core counts based on the specific sub-version (6.0, 6.1, 6.2), a unified L1/texture cache, and global/shared memory behaviors consistent with Compute Capability 5.x.
- [Compute Capability 7.x](../../../entities/cuda-compute-capability-7.x.md) — Compute Capability 7.x encompasses the Volta (7.0) and Turing (7.5) architectures, introducing Tensor Cores, Independent Thread Scheduling, and a unified data cache/shared memory partitioning system configurable via cudaFuncSetAttribute.
- [Compute Capability 8.x](../../../entities/cuda-compute-capability-8.x.md) — Compute Capability 8.x devices, part of the NVIDIA Ampere architecture, feature Streaming Multiprocessors (SMs) with varying core counts and unified data cache/shared memory configurations depending on the specific compute capability version.
- [Compute Capability 9.0](../../../entities/cuda-compute-capability-9.0.md) — Compute Capability 9.0 corresponds to the NVIDIA Hopper architecture, featuring advanced Streaming Multiprocessor (SM) designs with fifth-generation Tensor Cores supporting FP8 and other precision formats, a unified 256 KB data cache/shared memory hierarchy, and specialized matrix multiply-accumulate (MMA) acceleration features.
- [Compute Capability 10.0](../../../entities/cuda-compute-capability-10.0.md) — Compute Capability 10.0 corresponds to the first-generation Blackwell architecture, featuring fifth-generation Tensor Cores, a unified 100 KB data cache/shared memory, and specialized matrix multiply-accumulate (MMA) features extending from Hopper.
- [Compute Capability 12.0](../../../entities/cuda-compute-capability-12.0.md) — Compute Capability 12.0 is associated with the Blackwell architecture (second gen) and B200 devices, though official documentation for its specific architectural features is currently superseded by legacy Driver API references.
